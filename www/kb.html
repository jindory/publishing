Responsive Web & Hybrid App 2016.07

KB 락스타 웹/하이브리드 앱

  • Web/App
  • Blog
  • UI
  • UX
  • Publishing

대학생들을 위한 KB의 기업 블로그 사이트입니다. 디자인부터 퍼블리싱까지 참여하였습니다. 전체 프로젝트 일정이 촉박한 상황에 작업을 하게되어, 클라이언트의 의견반영과 타겟 그리고 프로젝트 일정까지 고려하여 작업을 해야하는 이슈가 있었습니다.

  • Index
  • 레이아웃 설정
  • 프로젝트에서 얻은 것
  • 작업중 아쉬운 내용 및 이슈

1. 레이아웃 설정

반응형 사이트에 하이브리드앱으로 작업될 예정이었기 때문에 사이트 디자인을 칼럼형식위주로 작업하여, 퍼블리싱 단계의 모바일화면 작업 공수를 줄일 수 있도록 하였습니다.

2. 프로젝트에서 얻은 것

협업툴과 커뮤니케이션

퍼블리셔, 프론트엔드, 백엔드 개발자분들과 Git이라는 툴을 사용하여 협업을 하게된 첫번째 프로젝트이고, 이 과정에서 다른 포지션의 분들과의 커뮤니케이션하는 법을 경험할 수 있었습니다.

상 수여

위 프로젝트는 ‘2016 대한민국 블로그 어워드’에서 대상을 수상하였습니다.

3. 작업중 아쉬운 내용 및 이슈

실 사용자의 니즈 부재에 대한 아쉬움

작업 후, gnb를 열어야만 페이지 이동이 되는게 불편하지 않냐는 질문을 받았습니다. kb락스타에서 사용자에게 제공하는 주요 서비스들은 꺼내서 메뉴를 제공하여서 큰 불편함이 없다고 생각했었지만, 실 사용자가 살펴봐야하는 내용은 뎁스를 한번 더 거쳐야하는 불편함이 있었습니다. 사이트의 목적에 따라 다르겠지만, 서비스 제공자가 사이트를 통해 사용자에게 안내하고싶은 내용과 실 사용자들이 필요한 정보의 중요도가 다를 수 있다는점을 한번 더 고민해볼 수 있 게 되었습니다.

2번의 뎁스가 필요한 페이지 이동

hover시 메뉴를 풀다운하여 펼쳐보도록 하거나, footer에서 사이트맵을 제공하여 전 페이지에서 이동이 용이하도록 하여야합니다.

운영 고려에 대한 아쉬움

메인과 서브페이지 작업을 진행할떄, 블로그 사이트의 운영에 따라 업로드가 자주 이루어질 것을 고려하여 작업 하였습니다. 하지만 메인 시안작업을 할때에, 클라이언트의 요구에 따라, 메인에 노출되는 컨텐츠를 일러스트로 배치하였습니다. 블로그 운영시에 불편할 것이라는걸 인식했지만.. 프로젝트 일정이 부족한 상황이어서 제안하지 못했습니다. 이 부분은 결국 관리자가 직접 등록할 수 있는 CMS로 수정된 것을 다음해에 확인했습니다.
